tailwindcss基本介绍

Tailwind CSS 是一个将**实用原子类(Atomic Utility First Class)**发挥到极致的 CSS 框架。

Tailwindcssv3.0将2.0 just in time模式设置为了默认项，现在他能够扫描规则配置下的所有文件以查找类名，生成静态css文件。

它快速、灵活且可靠——运行时间为零。

这种按需引入的特性使得我们必须注意：

1,不能拼接tailwindcss类名 e.g:mt-[${size === 'lg' ? '22px' : '17px' }]

2,不要动态计算任意值 e.g:bg-[]

3,不要使用空格 e.g:h-[calc(1000px - 4rem)] //删除空格，tailwind会处理calc自动加上空格

4,更具体的使用css变量 e.g :text-[color:var(–mystery-var)]

响应式配置

1,设置断点

tailwindcss使用屏幕断点作为前缀可以在屏幕宽度符合条件时覆盖原有样式。

实际开发时，需要遵循最小尺寸优先的策略。即先编写最小宽度的样式，在逐步扩大宽度的过程中使用带断点的前缀“打补丁”。使用这种模式大大降低了响应式开发的门槛，开发效率非常高。

值得一提的是，在3.0中添加了一个新max-*变体，可让您根据配置的断点应用最大宽度媒体查询。

常用样式及布局

在开发中的tailwind.config.js中 theme 部分使用 spacing 键来定制 Tailwind 的默认间距/大小比例。

默认情况下，间距比例由 padding、 margin、 width、 height、 maxHeight、 gap、 inset、 space 和 translate 核心插件继承。

如w-6=width:48px;

一些常用的原子类：

w-{size}宽度max-w-{size}size一般使用断点如md

h-{size}高度max-h-{size}元素的最大高度

m{t|r|b|l}-{size} 功能类控制元素一侧的外边距。

p{t|r|b|l}-{size} 功能类控制元素一侧的内边距。

gap-{x|y|' '}-{size}控制grid和flex间距。

display及position布局可以直省略
